# Sidecar moves

A sidecar is a bounded move that strengthens, attacks, or reframes an existing branch without adding abstract tree breadth. Log its evidence, contradictions, score changes, and result in `MAP.md`.

## Verification

- **Second-source check:** find an independent source for a load-bearing L1 item; confirm to reach L2 or open a contradiction.
- **Re-derivation:** recompute a derived figure from raw inputs another way.
- **Revision check:** look for later corrections or superseding versions of dated data.
- **Independence audit:** verify that purportedly independent sources do not share an origin.

## Adversarial checks

- **Steelman the runner-up:** make the strongest honest case for the second-ranked branch.
- **Assumption inversion:** flip the most load-bearing hypothesis and trace what breaks.
- **Base-rate check:** compare the answer with the relevant base rate and explain any exception.
- **Pre-mortem:** assume the chosen branch failed; identify likely causes and turn named gaps into frontier items.

## Reframes and frame escape

- **Constraint interrogation:** test whether each stated constraint is real or inherited; relax one only if it yields a materially different branch.
- **Objective swap:** test a different objective, such as robustness instead of speed; a ranking flip means the objective needs clarification.
- **Level shift:** ask whether the root question is a symptom of a question one level up; retain the original root and add the parent as a node.

Run a frame-escape check when the leading branch is blocked and weaker alternatives are only more desk-actionable, or when repeated work keeps refining one narrow frame. Re-read the root and constraints, run one reframe, and either record why the frame holds or add a materially better top-level branch. Do not reopen every old branch.

## Data access gate

Before a sidecar becomes the next action, record:

| Node | Required data | Direct path | Proxy path | Blocked if | Next move |
| --- | --- | --- | --- | --- | --- |
| N2.1 | call logs and outcomes | operator outreach | reviews, studies, public reports | neither path exists | proxy sidecar or switch branch |

An orientation sidecar sharpens a node but does not validate demand, feasibility, or viability by itself.
